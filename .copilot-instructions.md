# Instrucciones de Contexto para Copilot - React_RelatosPapel

## Arquitectura General

Este es un **monorepo con microservicios** basado en Spring Boot + API Gateway reactivo.

### Servicios principales
- **api-gateway** (puerto 8080): Gateway reactivo con patrón Phantom Token
- **ms-users** (puerto 8083): Microservicio de autenticación y gestión de usuarios
- **ms-catalogue** (puerto 8081): Catálogo de libros
- **ms-orders** (puerto 8082): Gestión de órdenes
- **eureka-server** (puerto 8761): Service Registry de NetFlix

## Autenticación: Patrón Phantom Token

### ✅ YA IMPLEMENTADO
La autenticación está **completamente implementada** con el patrón **Phantom Token**:

1. **Cliente → API Gateway:**
   - POST `/auth/login` con email/password
   - Recibe un `token opaco` (starts with `rp_opaque_`)

2. **Cliente → Servicios protegidos:**
   - Envía `Authorization: Bearer <opaqueToken>` 
   - API Gateway intercepta (filtro global `PhantomTokenGatewayFilter`)

3. **API Gateway → Redis:**
   - Intercambia el token opaco por JWT interno almacenado en Redis
   - Reescribe el header con el JWT

4. **Microservicio:**
   - Valida el JWT contra la clave pública almacenada localmente
   - Continúa el request si es válido

### Archivos clave de autenticación
- **ms-users/src/main/java/com/unir/papertales/users/service/TokenService.java**
  - Genera el token opaco
  - Almacena JWT en Redis
  
- **ms-users/src/main/java/com/unir/papertales/users/controller/AuthController.java**
  - POST `/api/v1/auth/login` → devuelve token opaco
  - POST `/api/v1/auth/register` → crea usuario nuevo
  - POST `/api/v1/auth/logout` → revoca token

- **api-gateway/src/main/java/com/unir/gateway/filter/PhantomTokenGatewayFilter.java**
  - Filtro global que intercambia token opaco por JWT
  - Solo aplica a rutas protegidas (`/catalogo/**`, `/ordenes/**`)

- **ms-catalogue/src/main/java/com/unir/papertales/catalogue/config/ResourceServerConfig.java**
  - Configura validación de JWT
  - Extrae roles del claim `roles`

## Endpoints de Autenticación

### Register (Nuevo)
```
POST /auth/register
Content-Type: application/json

{
  "email": "usuario@unir.com",
  "password": "password123",
  "role": "ROLE_LECTOR",  // opcional, default ROLE_LECTOR
  "enabled": true         // opcional, default true
}

Respuesta 201:
{
  "id": 3,
  "email": "usuario@unir.com",
  "role": "ROLE_LECTOR",
  "enabled": true
}
```

### Login
```
POST /auth/login
Content-Type: application/json

{
  "email": "usuario@unir.com",
  "password": "password123"
}

Respuesta 200:
{
  "accessToken": "rp_opaque_xxxx...",
  "tokenType": "Bearer",
  "expiresIn": 300
}
```

### Logout
```
POST /auth/logout
Authorization: Bearer rp_opaque_xxxx...

Respuesta 204 No Content
```

## Roles de usuario
- `ROLE_LECTOR`: Puede leer catálogo y órdenes
- `ROLE_ADMIN`: Puede crear/editar/borrar libros y órdenes

## Dependencias operativas
- **MySQL** en `localhost:3308` (ms-users)
  - Base: `db_users`
  - Tabla: `users`
  
- **Redis** en `localhost:6379`
  - Almacena mappeos: `opaqueToken → internalJWT`
  - Password: `relatos_secret_password`

- **Eureka** en `http://localhost:8761/eureka`
  - Registro dinámico de servicios

## Variables de entorno
```
EUREKA_URL=http://localhost:8761/eureka
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=relatos_secret_password
DB_URL=jdbc:mysql://localhost:3308/db_users
```

## Security Config
- CORS habilitado globalmente (todos los orígenes)
- CSRF deshabilitado (APIs REST)
- Session stateless (JWT)
- Rutas públicas sin autenticación:
  - `/auth/login`
  - `/auth/register`
  - `/actuator/**`

## Desarrollo futuro
Cuando necesites:
- Agregar nuevos endpoints protegidos: Usa `@PreAuthorize("hasAnyAuthority('ROLE_LECTOR', 'ROLE_ADMIN')")`
- Validar roles específicos: `@PreAuthorize("hasAuthority('ROLE_ADMIN')")`
- Acceder al usuario actual: `@AuthenticationPrincipal JwtAuthenticationToken token`
